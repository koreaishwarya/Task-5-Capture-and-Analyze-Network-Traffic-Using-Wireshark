# Task 5: Capture and Analyze Network Traffic Using Wireshark

## Objective

The objective of this task is to capture live network traffic using Wireshark and analyze different network protocols to understand how devices communicate over a network.

## Tools Used

* Wireshark
* Windows Operating System
* Internet Connection

## Task Description

Wireshark was used to capture network packets generated during normal internet activity. The captured packets were analyzed to identify common protocols and understand their functions within network communication.

## Steps Performed

### 1. Install Wireshark

Downloaded and installed Wireshark on the system.

### 2. Start Packet Capture

Selected the active network interface (Wi-Fi/Ethernet) and started capturing packets.

### 3. Generate Network Traffic

Performed various network activities such as:

* Browsing websites
* Searching on Google
* Sending ping requests

### 4. Stop Packet Capture

Stopped the capture after approximately one minute.

### 5. Analyze Captured Packets

Applied protocol filters to inspect different types of traffic.

#### DNS Filter

dns

#### TCP Filter

tcp

#### HTTP Filter

http

#### ICMP Filter

icmp

### 6. Export Packet Capture

Saved the captured traffic as a `.pcapng` file for further analysis and submission.

---

## Protocols Identified

### DNS (Domain Name System)

* Resolves domain names into IP addresses.
* Observed DNS query and response packets when accessing websites.

### TCP (Transmission Control Protocol)

* Provides reliable communication between devices.
* Observed TCP handshakes (SYN, SYN-ACK, ACK) and data transfer packets.

### HTTP/HTTPS

* Used for web communication.
* Observed requests and responses generated while browsing websites.

### ICMP (Internet Control Message Protocol)

* Used for network diagnostics.
* Observed ICMP Echo Requests and Echo Replies generated by ping commands.
## Conclusion

This task successfully demonstrated how to capture and analyze network traffic using Wireshark. Multiple protocols, including DNS, TCP, HTTP/HTTPS, and ICMP were identified and examined. The exercise provided practical experience in packet analysis and network monitoring, helping to build a better understanding of network communications and cybersecurity fundamentals.
